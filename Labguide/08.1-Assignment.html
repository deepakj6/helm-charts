Designing for deployment - Pages Microservice

In this lab we will look into an existing brown field pages microservice and deploy it to the K8s cluster. DevOps team has been be approached by the development team after completing their sprint to deploy their code to production. How will you deploy? What information do you need? What files do you expect to have in place in order to deploy the application? Think through all these pointers before attacking the problem.

Learning Outcomes

After completing the lab, you will be able to understand

  1. Deployment Architecture

  2. Design for deployment

  3. Deploy 2 tier application

Understanding the high-level distributed architecture

We will be focusing on the pages microservice in this lab. In the next lab we will deploy the category microservice

Development of the pages microservice

  1. Docker image of pages microservice is available in the image registry. How do we tackle designing the deployment architecture? Discuss this with your pairs? What inputs do you need?

  2. It is necessary to understand the configuration requirements and dependencies, if any, in order for the application to work properly once deployed.

  3. Let us examine the deployment architecture. The microservice is composed of a 2 tier architecture. In order to complete the design:

    1. We will have to design the db tier and service tier

    2. Look at the dependencies and figure out a way to connect the database tier with the service tier

Designing the architecture

  1. Delete the contents of ~/workspace/kubernetes-manifests directory

  2. Download pages microservice manifest files and extract to ~/workspace/kubernetes-manifests/pages

  3. Download mysql manifest files and extract to ~/workspace/kubernetes-manifests/mysql

  4. Walkthrough the manifest files & understand the solution to the deployment architecture.

  5. Before we start deploying, replace [student-name] with your namespace in all the manifest files.

Deploying to K8s

  1. Create storage related objects in minikube

    kubectl apply -f ~/workspace/kubernetes-manifests/mysql/storage-class.yaml
kubectl apply -f ~/workspace/kubernetes-manifests/mysql/pv.yaml
kubectl apply -f ~/workspace/kubernetes-manifests/mysql/pvc.yaml
    kubectl get storageclasses
kubectl get pv
kubectl get pvc

  2. Create configmaps,secrets and rbac

    kubectl apply -f ~/workspace/kubernetes-manifests/mysql/configmap.yaml
kubectl apply -f ~/workspace/kubernetes-manifests/mysql/secret.yaml
kubectl apply -f ~/workspace/kubernetes-manifests/pages/config.yaml
kubectl apply -f ~/workspace/kubernetes-manifests/pages/rbac.yaml

  3. Create all the services

    kubectl apply -f ~/workspace/kubernetes-manifests/mysql/service.yaml
kubectl apply -f ~/workspace/kubernetes-manifests/pages/service.yaml

  4. Create all the deployments and job in minikube

    kubectl apply -f ~/workspace/kubernetes-manifests/mysql/deployment.yaml
kubectl apply -f ~/workspace/kubernetes-manifests/mysql/flyway-job.yaml
kubectl apply -f ~/workspace/kubernetes-manifests/pages/deployment.yaml

  5. We successfully deployed pages microservice application to K8s cluster. But, does it work as expected? Test it using basic curl commands or postman

  6. Refer Curl Guide for curl testing guide

  7. Does GET request work?

  8. Does POST request work?

  9. Were you able to find any defects in this version of the deployment? Note down any problems that you might encounter and discuss them with your pairs to find suitable solutions
